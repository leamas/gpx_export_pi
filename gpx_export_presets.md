# GPX Export Plugin Presets

This document describes the built-in GPX compatibility presets in the OpenCPN GPX Export plugin. The preset definitions are in `src/gpx_preset.cpp`, and the fields actually written to the GPX file are controlled by `src/gpx_writer.cpp`.

## How the presets are used

The plugin builds an internal `ExportDocument` containing selected routes, route points, waypoints, and, in the model, tracks. GPX output is generated by passing that document plus a `GpxCompatibilityPreset` to `gpx_writer::Generate()`.

All built-in presets currently write GPX 1.1 and produce this common structure:

```xml
<gpx version="1.1" creator="OpenCPN GPX Export Plugin">
  <wpt>...</wpt>
  <rte>
    <name>...</name>
    <rtept>...</rtept>
  </rte>
  <trk>
    <trkseg>
      <trkpt>...</trkpt>
    </trkseg>
  </trk>
</gpx>
```

The exported object type depends on the user selection in the dialog. The current UI supports route and waypoint selection. The writer also has support for tracks if `ExportDocument.tracks` is populated in the future.

## Common GPX fields

### Waypoints and route points

For each standalone waypoint (`<wpt>`) and each route point (`<rtept>`), the writer always includes:

- `lat` attribute
- `lon` attribute
- `<name>`

Depending on the preset, it may also include:

- `<time>` when timestamps are enabled and a timestamp exists
- `<desc>` when descriptions are enabled and a description exists
- `<sym>` when symbols are enabled and a symbol exists
- `<type>` when OpenCPN extensions are enabled and a waypoint type exists
- `<extensions>` with OpenCPN waypoint extension data when OpenCPN extensions are enabled

### Routes

For each route (`<rte>`), the writer includes:

- `<name>` when the route name is not blank
- `<rtept>` entries for the route points

Depending on the preset, it may also include:

- `<desc>` when descriptions are enabled and a route description exists
- `<extensions>` with OpenCPN and/or Garmin route extension data when extensions are enabled

### Tracks

For each track (`<trk>`), the writer includes:

- `<name>` when the track name is not blank
- `<trkseg>`
- `<trkpt lat="..." lon="...">`

Depending on the preset, it may also include:

- `<desc>` when descriptions are enabled and a track description exists
- `<time>` on track points when timestamps are enabled and a timestamp exists

The current writer does not export track speed-over-ground (`sog_knots`) or course-over-ground (`cog_degrees`), even though those fields exist in the internal model.

## Extension fields

Only the **OpenCPN Full-Fidelity GPX** preset enables extension output by default.

### OpenCPN waypoint extensions

When OpenCPN waypoint extensions are enabled, the writer can include:

- `<type>` outside the extension block, currently usually `WPT`
- `<opencpn:guid>`
- `<opencpn:viz_name>`
- `<opencpn:shared>`
- `<opencpn:arrival_radius>`
- `<opencpn:waypoint_range_rings visible="..." number="..." step="..." units="..." colour="..." />`
- `<opencpn:scale_min_max UseScale="..." ScaleMin="..." ScaleMax="..." />`

A waypoint extension block is only written when at least one of the OpenCPN extension conditions is present, such as a GUID, name visibility flag, shared flag, arrival radius, range rings, or scale settings.

### OpenCPN route extensions

When OpenCPN route extensions are enabled, the writer can include:

- `<opencpn:guid>`
- `<opencpn:viz>`
- `<opencpn:start>`
- `<opencpn:end>`
- `<opencpn:planned_speed>`
- `<opencpn:planned_departure>`
- `<opencpn:time_display>`

### Garmin route extensions

When Garmin extensions are enabled, the writer includes:

- `<gpxx:RouteExtension>`
- `<gpxx:IsAutoNamed>`

## Preset details

## Generic Marine GPX

**Preset ID:** `generic_marine`

**Purpose:** Safe general-purpose GPX output for marine chartplotters and applications that can handle standard GPX 1.1 but do not need vendor-specific extensions.

**Included fields:**

- GPX 1.1 root element
- Waypoint and route-point latitude/longitude
- Waypoint and route-point names
- Waypoint and route-point timestamps when present
- Waypoint and route-point descriptions when present
- Route name when present
- Route description when present
- Track name when present
- Track description when present
- Track-point timestamps when present

**Excluded fields:**

- Symbols/icons (`<sym>`)
- GPX extension blocks
- OpenCPN extension fields
- Garmin extension fields
- Duplicate top-level waypoint copies of route points

**Naming behavior:**

- Preserves original waypoint and route-point names.
- Requires names to be non-blank during validation.
- Requires exported waypoint names to be unique within each waypoint batch or route-point batch.
- Does not strip special characters.
- Does not enforce a maximum name length.

## OpenCPN Full-Fidelity GPX

**Preset ID:** `opencpn_full`

**Purpose:** Highest-fidelity export intended to preserve as much OpenCPN-specific information as the plugin can access. This is the best preset for round-tripping back into OpenCPN or preserving OpenCPN metadata.

**Included fields:**

- GPX 1.1 root element
- OpenCPN namespace declaration: `xmlns:opencpn="http://www.opencpn.org"`
- Garmin GPX extension namespace declaration: `xmlns:gpxx="http://www.garmin.com/xmlschemas/GpxExtensions/v3"`
- Garmin extension schema location
- Waypoint and route-point latitude/longitude
- Waypoint and route-point names
- Waypoint and route-point timestamps when present
- Waypoint and route-point descriptions when present
- Waypoint and route-point symbols/icons when present
- Waypoint and route-point type, normally `WPT`
- OpenCPN waypoint extensions when relevant
- Route name when present
- Route description when present
- OpenCPN route extensions
- Garmin route extension containing `gpxx:IsAutoNamed`
- Track name when present
- Track description when present
- Track-point timestamps when present

**OpenCPN waypoint/route-point extension fields:**

- GUID
- Name visibility flag
- Shared route-point flag
- Arrival radius
- Range ring visibility/count/spacing/units/color
- Scale minimum/maximum settings

**OpenCPN route extension fields:**

- Route GUID
- Route visibility
- Route start label
- Route end label
- Planned speed, when known
- Planned departure, when known
- Time display string

**Excluded fields:**

- Duplicate top-level waypoint copies of route points

**Naming behavior:**

- Preserves original waypoint and route-point names.
- Does not require names to be unique.
- Does not require names to be non-blank.
- Does not strip special characters.
- Does not enforce a maximum name length.

## Minimal GPX

**Preset ID:** `minimal`

**Purpose:** Very small, conservative GPX output with only essential position and name data. This is useful for systems that are sensitive to optional GPX fields or vendor extension data.

**Included fields:**

- GPX 1.1 root element
- Waypoint and route-point latitude/longitude
- Waypoint and route-point names
- Route name when present
- Track name when present

**Excluded fields:**

- Timestamps
- Descriptions
- Symbols/icons
- GPX extension blocks
- OpenCPN extension fields
- Garmin extension fields
- Duplicate top-level waypoint copies of route points
- Track-point timestamps

**Naming behavior:**

- Does not preserve original waypoint or route-point names.
- Numbers waypoint and route-point names sequentially as `001`, `002`, `003`, etc.
- Requires names to be non-blank during validation.
- Requires exported waypoint names to be unique within each waypoint batch or route-point batch.
- Because generated names are numeric, the exported names contain no special characters.
- Does not enforce a maximum name length.

## Important implementation notes

### 1. Top-level `strip_special_chars` is not currently applied by the writer

The `GpxCompatibilityPreset` struct has both:

```cpp
bool strip_special_chars;
NamePolicy name_policy;
```

However, the writer applies only `preset.name_policy` through `name_policy::ApplyPolicyBatch()`. It does not directly use `preset.strip_special_chars`.

That means these preset assignments in `gpx_preset.cpp` are not applied during a direct export unless the user opens the Options dialog and saves the options, because the Options dialog copies the top-level flag into `name_policy.strip_special_chars`:

```cpp
p.strip_special_chars = true;
```

This affects at least:

- Garmin Safe GPX
- Furuno Safe GPX

It does not practically affect Minimal GPX or Raymarine Safe GPX because those presets generate numeric names (`001`, `002`, etc.), which contain no special characters anyway.

Recommended fix:

```cpp
// In each preset that intends to strip special characters:
p.strip_special_chars = true;
p.name_policy.strip_special_chars = true;
```

Or, alternatively, remove the duplicated top-level `strip_special_chars` field and use only `NamePolicy::strip_special_chars`.

### 2. Route names are not transformed by the name policy

The name policy is applied to standalone waypoint names and route-point names. The writer does not apply the name policy to route names or track names. Route and track names are written as-is, apart from XML escaping.

### 3. Name uniqueness is scoped separately

Uniqueness is applied separately to:

- the top-level waypoint list
- each route's route-point list

A standalone waypoint and a route point can still end up with the same name because they are processed in separate batches.

### 4. Route points are not exported as top-level waypoints by any built-in preset

The preset field `include_route_points_as_top_level_wpts` exists, and the writer supports it, but all built-in presets currently set it to `false`.

### 5. OpenCPN Full-Fidelity GPX does not validate blank or duplicate names

This is intentional based on the preset settings. It maximizes fidelity to the original OpenCPN data rather than forcing compatibility restrictions.

### 6. `max_name_length` is duplicated

The preset has both:

```cpp
std::optional<size_t> max_name_length;
NamePolicy::max_length;
```

The writer only enforces `NamePolicy::max_length`. Furuno Safe GPX sets both, so the 20-character limit is applied. For future presets, set `name_policy.max_length` if the writer should enforce the limit.

## Recommended preset usage

Use **OpenCPN Full-Fidelity GPX** when exporting for backup, troubleshooting, or re-import into OpenCPN.

Use **Generic Marine GPX** as the default for broad compatibility when descriptions and timestamps are useful.

Use **Minimal GPX** when a receiving device is very strict or fails to import richer GPX files.

Use **Garmin Safe GPX**, **Raymarine Safe GPX**, **B&G / Navico Safe GPX**, or **Furuno Safe GPX** when targeting those systems and prioritizing compatibility over metadata preservation.
